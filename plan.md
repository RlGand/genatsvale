# План разработки: сайт «Ресторан Генацвале на Мира»

Учебный проект (практика колледжа). Вёрстка по макету Figma (`figma/`), адаптация под нормальные размеры элементов.

---

## 1. Текущее состояние

| Что есть | Статус |
|----------|--------|
| `index.html` | Заготовка: цитата, подключены шрифты Poppins / ABeeZee |
| `css/style.css` | Минимум (цвет заголовка) |
| `img/` | Логотип, фон, фото блюд, ресторан, хинкали, стрелки |
| `figma/` | 4 кадра макета (экспорт JPG) |

**Проблема макета:** элементы в Figma слишком крупные («как для слепых»). Frame 1 экспортирован в **5760×21176 px** (≈4× от стандартной ширины 1440 px). Остальные кадры — **1440 px** по ширине. При вёрстке ориентируемся на **логическую ширину 1440 px**, а в CSS уменьшаем масштаб примерно на **15–20%** (коэффициент **0,82–0,85**).

## 3. Дизайн-система (зафиксировать в CSS-переменных)

### Цвета (по макету)

```css
:root {
  --color-header-footer: #4a2c2a;   /* тёмно-коричневый шапка/подвал — уточнить пипеткой из Figma */
  --color-bg-main: #f5f0e8;         /* светлый бежевый фон контента */
  --color-card: #c4a484;            /* терракотовые карточки */
  --color-text-dark: #3d2b1f;
  --color-text-light: #ffffff;
  --color-button: #8b6f5c;
  --scale: 0.85;                    /* глобальное уменьшение от макета */
}
```

Точные HEX — снять с макета в Figma или пипеткой по JPG перед финальной сдачей.

### Шрифты

- **Декоративный** (логотип, навигация, подвал): в макете — «этнический» гротеск; в проекте пока Poppins — **подобрать ближайший** (или экспорт текста как SVG / подключить шрифт из Figma, если есть).
- **Основной текст** (заголовки категорий, форма): чистый sans / лёгкий serif — уточнить по макету; ABeeZee как запасной вариант для подписей.

### Масштабирование (главное требование)

| Элемент макета (условно) | После `scale: 0.85` |
|--------------------------|---------------------|
| Ширина макета 1440 px | `max-width: 1224px` (1440 × 0,85) |
| Карточка категории ~400 px | ~340 px |
| Круглое фото блюда ~280 px | ~238 px |
| Заголовок категории 32 px | ~27 px |
| Кнопка «Заказать» высота ~56 px | ~48 px |
| Отступы секций 80 px | ~68 px |

Дополнительно для **десктопа**: сетка категорий **3–4 колонки** вместо 2 из макета — страница короче, элементы визуально меньше без потери читаемости.

├── figma/                  # референс (4 JPG)

Используется **один `style.css`**, разбитый на секции комментариями.

## 7. Риски и решения

| Риск | Решение |
|------|---------|
| Нет шрифта из макета | Google Fonts + ближайший аналог или экспорт заголовков SVG |
| Не хватает фото категорий | Временные плейсхолдеры из `img/eda.png`, замена по мере готовности |
| Frame 1 слишком тяжёлый | Верстать по секциям из Frame 2–4 + отдельный скриншот hero из Figma 1× |
| Преподаватель требует точный пиксель-perfect | Уменьшить только `scale`, не менять пропорции; задокументировать коэффициент 0,85 в отчёте |
